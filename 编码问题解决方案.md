# 控制台乱码问题解决方案

## 问题描述
在 Windows 系统上运行 Electron 应用时，控制台输出的中文会显示为乱码。这是因为 Windows 命令行默认使用 GBK 编码，而 Node.js/Electron 使用 UTF-8 编码。

## 解决方案

### 方法1：使用批处理文件启动（推荐）
双击以下文件启动应用：
- `start.bat` - 正常启动应用
- `dev.bat` - 开发模式启动

这些批处理文件会自动设置控制台编码为 UTF-8。

### 方法2：使用 PowerShell 脚本
在 PowerShell 中运行：
```powershell
.\start.ps1
```

### 方法3：手动设置编码后启动
在命令行中执行：
```bash
chcp 65001
npm start
```

### 方法4：直接使用 npm 脚本
package.json 中的脚本已经包含编码设置：
```bash
npm start    # 自动设置编码
npm run dev  # 开发模式，自动设置编码
```

## 技术细节

1. **encoding-fix.js** - 编码修复模块，在应用启动时自动设置正确的编码
2. **main.js** - 已在文件开头引入编码修复模块
3. **批处理文件** - 使用 `chcp 65001` 命令设置控制台代码页为 UTF-8

## 注意事项

- 如果使用 VSCode 的集成终端，通常不会有乱码问题
- Windows Terminal 也默认支持 UTF-8，不会有乱码
- 传统的 cmd.exe 需要手动设置编码

## 测试方法

运行以下命令测试编码是否正确：
```bash
node encoding-fix.js
```

如果能正确显示中文，说明编码设置成功。
